# Simulação de Fluxo de Trabalho de QA

## Visão Geral

Este repositório contém uma simulação documentada do dia a dia e do fluxo de trabalho de um Analista de Qualidade (QA) em um projeto de software. A simulação é baseada no desenvolvimento de uma nova funcionalidade ("Cupom de Desconto") para um e-commerce fictício chamado "Swagglag Shopping".

O objetivo é demonstrar de forma prática o ciclo de vida de testes de software (STLC), desde a análise de requisitos até a aprovação final para produção.

## Estrutura do Repositório e Artefatos

Os documentos abaixo representam os artefatos gerados durante o ciclo de vida de testes desta funcionalidade. Eles estão organizados na ordem em que foram criados e devem ser lidos.

1.  **[FLUXO_DE_TRABALHO_QA.md](FLUXO_DE_TRABALHO_QA.md)**
    - **Propósito:** O documento mestre que define a metodologia, os processos e os padrões de QA a serem seguidos no projeto.

2.  **[PLANO_DE_TESTES_CUPOM.md](PLANO_DE_TESTES_CUPOM.md)**
    - **Propósito:** O plano de testes específico para a funcionalidade de cupom. Contém a análise de escopo e os casos de teste detalhados (positivos, negativos e de UI).

3.  **[EXECUCAO_TESTES_CUPOM.md](EXECUCAO_TESTES_CUPOM.md)**
    - **Propósito:** O log de execução que serve como evidência do trabalho de teste. Registra os resultados de cada caso de teste, incluindo a descoberta de um bug e a posterior rodada de testes de regressão e confirmação.

4.  **[BUG_REPORT_001.md](BUG_REPORT_001.md)**
    - **Propósito:** O relatório formal do bug encontrado durante a fase de execução. Este documento detalha o problema de forma clara para que a equipe de desenvolvimento possa atuar na correção.

## Como Navegar
Recomenda-se seguir a ordem numérica dos arquivos para entender o fluxo completo, desde a definição do processo geral até a resolução de um problema específico.
